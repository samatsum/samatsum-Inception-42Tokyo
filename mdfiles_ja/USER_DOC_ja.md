# ユーザー向けドキュメント - Inception インフラストラクチャ

このガイドでは、Inceptionインフラストラクチャが提供するサービスの利用方法および管理方法について説明します。

## 1. サービスへのアクセス

すべてのサービスは、TLSを用いたポート443経由でNGINXリバースプロキシによってルーティングされています。ブラウザでドメイン `samatsum.42.fr` を使用してアクセスしてください。

| サービス | URL | 説明 |
| :--- | :--- | :--- |
| **WordPress** | `https://samatsum.42.fr` | メインのウェブサイトおよびブログプラットフォーム。 |
| **Adminer** | `https://samatsum.42.fr/adminer` | MariaDB用のデータベース管理ツール。 |
| **静的サイト** | `https://samatsum.42.fr/static` | 軽量なポートフォリオ/履歴書サイト。 |
| **Grafana** | `https://samatsum.42.fr/grafana` | インフラ監視ダッシュボード。 |

*注意: 自己署名SSL証明書を使用しているため、ブラウザにセキュリティ警告が表示されます。「詳細設定」から「samatsum.42.fr に進む」を選択して続行してください。*

## 2. 認証情報とログイン

セキュリティ上の理由から、認証情報はリポジトリ内には保存されていません。

- **WordPress管理者**: 主要な管理者アカウント（'admin'を含まない名前）。パスワードは `secrets/credentials.txt` を確認してください。
- **WordPress一般ユーザー**: 標準ユーザーアカウント。`secrets/wp_normal_password.txt` を確認してください。
- **データベース (MariaDB)**: Adminer経由でログインする際は、`secrets/db_password.txt` にある認証情報を使用してください。
- **FTPアクセス**: ポート21でFTPクライアントを使用して接続する際は、`secrets/ftp_password.txt` のパスワードを使用してください。

## 3. 日常の運用操作 (Makefile)

標準的な操作は、ルートディレクトリにある `Makefile` を使用して行います。

- **サービスの起動**: `sudo make up` （すべてのコンテナをバックグラウンドでビルド・起動）。
- **ステータス確認**: `make status` （実行中のコンテナ一覧と状態を表示）。
- **ログの表示**: `make logs` （トラブルシューティング用のリアルタイムログを出力）。
- **サービスの停止**: `make stop` （データとコンテナ状態を維持したまま、プロセスを安全に停止）。
- **サービスの再開**: `make start` （停止状態からサービスを再開）。

## 4. ファイル転送 (FTP)

WordPressサイトに直接ファイルをアップロードまたは変更する場合：
1. FTPクライアント（FileZilla等）を開きます。
2. ホスト: `samatsum.42.fr` | ポート: `21`。
3. `.env` で指定されたユーザー名と `secrets/ftp_password.txt` のパスワードを入力します。
4. `/var/www/html` ディレクトリへアクセスが可能になります。
